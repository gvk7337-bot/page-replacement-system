// How to Read the Results

Page Faults

A page fault happens when the CPU requests a page not in RAM. The OS must fetch it from disk — which is slow. Fewer faults means better performance.

Hit Ratio

Hit Ratio = (Hits / Total References) x 100%. A higher hit ratio means the algorithm keeps the most useful pages in memory.

Why Optimal Always Wins

Optimal knows all future references in advance, which is impossible in a real OS. It is a theoretical benchmark showing the minimum achievable fault count.

Effect of Frame Count

More frames usually means fewer faults. However FIFO can show Belady's Anomaly: more frames can sometimes cause more faults. LRU and Optimal are immune.
