Algorithm Theory

Concepts, worked examples, comparison table, and FAQ for FIFO, LRU, and Optimal page replacement algorithms.

Background

What is Page Replacement?

Virtual Memory and Paging

Modern operating systems use virtual memory to give each process the illusion of a large, contiguous address space. Physical RAM is divided into fixed-size units called frames, and a process's address space is divided into equally sized units called pages.

What is a Page Fault?

When the CPU requests a page not currently in RAM, a page fault occurs. The OS must fetch that page from disk and load it into a free frame. If no frame is free, it must evict an existing page to make room — this is where the replacement algorithm is applied.

Key Terms

  • Frame: A slot in physical RAM that holds one page.
  • Page: A fixed-size chunk of a process's virtual address space.
  • Page Fault: Accessing a page that is NOT in RAM (miss).
  • Page Hit: Accessing a page that IS already in RAM.
  • Hit Ratio: (Total Hits / Total References) x 100%. Higher is better.
  • Reference String: The ordered sequence of page numbers a process accesses.

FIFO

First-In, First-Out

Concept

Replace the page that was loaded into memory first — the oldest page in RAM. Imagine a queue at a counter: whoever arrived first leaves first.

How It Works

  • Maintain a circular pointer over the frames.
  • On a page fault: load the new page at the pointer position.
  • Advance the pointer using modulo arithmetic.
  • On a page hit: do nothing, pointer stays.

Worked Example

Ref: 7,0,1,2,0,3,0,4  |  Frames: 3

Step 1: Ref=7 | [7,-,-] | FAULT
Step 2: Ref=0 | [7,0,-] | FAULT
Step 3: Ref=1 | [7,0,1] | FAULT
Step 4: Ref=2 | [2,0,1] | FAULT (7 out, oldest)
Step 5: Ref=0 | [2,0,1] | HIT
Step 6: Ref=3 | [2,3,1] | FAULT (0 out)
Step 7: Ref=0 | [2,3,0] | FAULT (1 out)
Step 8: Ref=4 | [4,3,0] | FAULT (2 out)

Faults: 7  |  Hits: 1  |  Ratio: 12.5%

Advantages

  • Extremely simple to implement.
  • Very low overhead (just one integer pointer).

Disadvantages

  • Ignores how recently or frequently a page is used.
  • Suffers from Belady's Anomaly: more frames can sometimes cause more faults.
  • Not used in modern OS due to poor performance.
LRU

Least Recently Used

Concept

Replace the page that has not been used for the longest time. Based on temporal locality: recently used pages are likely to be used again soon.

How It Works

  • Track a timestamp (step number) for each frame's last access.
  • On a page fault: evict the frame with the smallest (oldest) timestamp.
  • On a page hit: update that frame's timestamp to the current step.

Worked Example

Ref: 7,0,1,2,0,3,0,4  |  Frames: 3

Step 1: Ref=7 | [7,-,-] | FAULT (7@1)
Step 2: Ref=0 | [7,0,-] | FAULT (0@2)
Step 3: Ref=1 | [7,0,1] | FAULT (1@3)
Step 4: Ref=2 | [2,0,1] | FAULT (7 LRU@1)
Step 5: Ref=0 | [2,0,1] | HIT (0@5)
Step 6: Ref=3 | [2,0,3] | FAULT (1 LRU@3)
Step 7: Ref=0 | [2,0,3] | HIT (0@7)
Step 8: Ref=4 | [4,0,3] | FAULT (2 LRU@4)

Faults: 6  |  Hits: 2  |  Ratio: 25%

Advantages

  • Better performance than FIFO in practice.
  • Immune to Belady's Anomaly (stack algorithm).
  • Exploits temporal locality effectively.

Disadvantages

  • Requires tracking access history for all pages.
  • More implementation overhead than FIFO.
OPT

Optimal (Belady's)

Concept

Replace the page whose next use is farthest in the future. If a page will never be used again, it is the best candidate. Guarantees the minimum possible fault count.

How It Works

  • On a page fault: look ahead in the reference string.
  • For each frame, find the next occurrence of its page.
  • Evict the frame whose page appears latest (or never) in the future.

Worked Example

Ref: 7,0,1,2,0,3,0,4  |  Frames: 3

Step 1: Ref=7 | [7,-,-] | FAULT
Step 2: Ref=0 | [7,0,-] | FAULT
Step 3: Ref=1 | [7,0,1] | FAULT
Step 4: Ref=2 | [2,0,1] | FAULT (7 never used again)
Step 5: Ref=0 | [2,0,1] | HIT
Step 6: Ref=3 | [2,0,3] | FAULT (1 used never again)
Step 7: Ref=0 | [2,0,3] | HIT
Step 8: Ref=4 | [4,0,3] | FAULT (2 used never again)

Faults: 5  |  Hits: 3  |  Ratio: 37.5%

Advantages

  • Provably optimal: minimum possible faults.
  • Immune to Belady's Anomaly.
  • Perfect benchmark for comparing real algorithms.

Disadvantages

  • Cannot be implemented in a real OS (future is unknown).
  • Only useful for simulation and analysis.

Note

Belady's Anomaly — FIFO Specific

With FIFO, increasing the number of frames can sometimes increase the number of page faults. This is counterintuitive and is called Belady's Anomaly.

Classic demonstration:
Reference String: 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5

With 3 frames (FIFO): 9 page faults
With 4 frames (FIFO): 10 page faults  ←  more frames caused MORE faults!

LRU and Optimal are called stack algorithms. For stack algorithms, adding one more frame can only decrease or maintain the fault count — it can never increase it. This is why LRU and Optimal do not suffer from Belady's Anomaly.

FAQ

Frequently Asked Questions

A page hit occurs when the CPU requests a page that is already loaded in one of the RAM frames — no disk access is needed, and execution continues quickly. A page fault occurs when the requested page is NOT in RAM. The OS must pause the process, fetch the page from disk (which is thousands of times slower than RAM), load it into a frame, and then resume execution. Minimizing page faults is the primary goal of page replacement algorithms.
The Optimal algorithm has complete knowledge of all future page references. When it must evict a page, it always picks the one that will not be needed for the longest time — or will never be needed again. This is mathematically proven to produce the minimum possible fault count for any given input. No other algorithm can do better, because every other eviction choice would risk faulting sooner. This is why Optimal is used as a theoretical benchmark — you compare FIFO and LRU against it to see how close they get.
Yes, but not in its pure form. True LRU requires recording the exact time of every memory access, which adds significant hardware and software overhead. Real operating systems use approximations of LRU, such as the Clock (Second-Chance) algorithm or the NFU (Not Frequently Used) algorithm. These give LRU-like behavior at a much lower cost. The core idea — keeping recently used pages in memory — is widely used in modern OS page management.
Belady's Anomaly is the counterintuitive phenomenon where adding more memory frames causes MORE page faults with the FIFO algorithm. You would normally expect that more RAM means better performance, but FIFO's simplistic replacement strategy does not guarantee this.

Only FIFO (and a few other non-stack algorithms) can exhibit this anomaly. LRU and Optimal are called stack algorithms, which means: if a page is in memory with N frames, it will also be in memory with N+1 frames. This property guarantees they are immune to Belady's Anomaly.
Thrashing occurs when a process spends more time handling page faults than executing actual instructions. This happens when a process does not have enough frames to hold its working set — the set of pages it actively uses. When thrashing occurs, the CPU is almost always waiting for disk I/O, and system performance collapses. A good page replacement algorithm (like LRU) reduces thrashing by keeping the most actively used pages in memory. Increasing the number of frames is the most direct fix.
The formula is: Hit Ratio = (Total Page Hits / Total Page References) × 100%

A higher hit ratio is always better. In this simulator, hit ratio directly shows how often the algorithm avoided a costly disk fetch. In real systems, modern OS memory managers aim for a hit ratio of 99% or higher for well-behaved workloads. On small examples with few frames (like the 3-frame, 8-page demos here), ratios of 10–40% are normal because there are very few frames available.
The Working Set Model, introduced by Peter Denning, defines the working set W(t, Δ) as the set of pages accessed during the last Δ memory references (the working set window). The idea is that a process needs all its working-set pages in memory to run efficiently. If a process is allocated fewer frames than its working set size, it will thrash. The model guides OS decisions about how many frames to give each process — it is one of the most important concepts in memory management beyond basic page replacement.
For exam purposes: Optimal produces the fewest faults but cannot be implemented. LRU is the best practical algorithm because it approximates Optimal well using past access history, does not suffer from Belady's Anomaly, and is widely used in real OS implementations (via approximations like the Clock algorithm). FIFO is the simplest but performs worst and can suffer from Belady's Anomaly. A good exam answer acknowledges all three roles: FIFO for simplicity, LRU for practice, Optimal for benchmarking.
